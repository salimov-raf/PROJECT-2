# <center> Проект 2: Анализ резюме из HeadHunter (SQL)  </center>


# 1. Краткое описание задачи.

Имеется база резюме, реализованная в составе нескольких таблиц, находящихся в схеме public базы данных project_sql.  
Необходимо провести (согласно заданий):  
- предварительный анализ данных;
- детальный анализ вакансий;
- анализ работодателей;
- предметный анализ;
- дополнительное исследование (по выболру).


# 2. Исходные данные.

2.1. Для исследования предоставлены следующие таблицы:

- vacancies - Хранит в себе данные по вакансиям
- areas - Справочник, который хранит код региона и его название
- employers - Справочник со списком работодателей
- industries - Справочник вариантов сфер деятельности работодателей
- employers_industries - Дополнительная таблица, которая существует для организации связи между работодателями и сферами их деятельности  

2.2.  Таблицы связаны между собой согласно следующей схемы:

# 3. Краткие пояснения общего характера 
3.1. Запрос SQL имеет структуру:  
<center> "query_X_Y" </center> <br>


Числовое значение или DataFrame, содержащий ответ имеет структуру:  

<center> "answer_X_Y"  </center> <br>

Здсеь:  
- первая цифра (Х) - номер раздела (юнита), 
- вторая цифра (Y) - номер задания в юните

3.2 В юнитах 3-5 включительно ко всем заданиям для самопроверки выполнял задание методами работы с DataFrame. Ячейка с таким кодом расположена сразу за основным решением и начинается со слов  

<center>"# ПРОВЕРКА_X_Y1"</center> <br>

а решение находится в DataFrame с наименованием 

<center>"answer_X_Y1"</center> <br>

Для реализации этого я заранее сохранил таблицы в файлы .cvs с соответствующими наименованиями (код не привожу), а в Юните 3 создаю DataFram'ы с этими же именами. Файлы .cvs лежат в папке data.

3.3. Покольку GitHub не поддерживает отображение интерактивной визуализации в Plotly, то все рисунки и графики собраны в папку images, причем графики представлены в html-формате. Строки, которые создавали эти графики, оставлены в соответствующих ячейках ноутбука, но переведены в комментарии
